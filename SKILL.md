# 项目执行约束

## 适用范围

本文件约束本项目后续所有开发 Batch。执行 Agent 必须先阅读 `docs/` 下的项目定位、产品范围、Batch 计划、验收清单、UI 风格和技术边界，再开始对应 Batch。

## 执行规则

1. 每次只执行用户明确确认的 Batch，不提前进入下一批次。
2. 只实现当前 Batch 的验收项，保持最小改动，不扩大需求。
3. 不假设尚未实现的功能存在，不破坏已经通过的 Batch。
4. 技术选择必须服务于轻量、Windows、本地优先的桌面浮窗工具。
5. 未经项目文档和用户确认，不新增技术、模块或产品边界。
6. 每个 Batch 完成后先按 `docs/03_ACCEPTANCE_CHECKLIST.md` 自检，再停止并等待用户确认。

## 永久范围禁区

第一版明确禁止：AI 自动整理和 AI 每日复盘、行情功能或行情 API、荐股、券商同步或券商 API、自动交易、独立图片库、人工总结模块。不得以基础设施、预留接口或实验功能为由绕过这些边界。

## Batch 0 特别约束

Batch 0 只允许建立文档和空目录说明，不实现 UI、浮窗、数据库、AI 调用、行情接口、自动交易或其他业务逻辑。完成后必须停止，不进入 Batch 1。
