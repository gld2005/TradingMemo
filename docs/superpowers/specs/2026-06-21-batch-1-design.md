# Batch 1 项目骨架与 UI 基础设计

## 范围

本设计只覆盖 Batch 1：建立可运行的 Windows 桌面应用骨架、主窗口静态页面路由、浮窗静态占位入口、基础组件与明暗主题。不得实现笔记保存、图片处理、持久化、分类管理、搜索筛选、导出备份或任何外部服务。

## 技术路线

首选路线为 Tauri + React + TypeScript + Vite。当前开发环境缺少 Rust 与 Cargo，无法初始化和验证 Tauri，因此按任务允许的降级规则采用 Electron + React + TypeScript + Vite。只引入 Electron、React、Vite、TypeScript、Vitest 与测试所需的轻量依赖，不使用状态管理库、路由库、UI 组件库或图标库。

## 架构

- Electron 主进程创建主窗口和一个静态浮窗占位窗口。
- Vite 通过 URL 查询参数区分主窗口与浮窗入口，两个窗口共享同一套主题变量和基础组件。
- React 仅使用组件内部状态保存当前导航和主题；页面刷新或应用退出后无需保留。
- 主窗口由侧栏与内容区组成；内容区显示四个静态页面之一。
- 浮窗仅展示标题、输入占位、分类占位和禁用保存按钮，不具备桌面浮窗行为或数据能力。

## 组件与页面

基础组件包括 Button、Card、Input、EmptyState、Sidebar 和 PageHeader。页面包括今日记录、知识库、分类和设置，均只显示用户指定的空状态文案。布局使用统一 CSS 变量控制颜色、间距、圆角、边框和阴影。

## 状态与交互

- 导航点击只切换内存中的当前页面。
- 主题按钮只切换根节点的 `data-theme` 属性，不写入文件、数据库或浏览器存储。
- 浮窗保存按钮保持禁用，并通过辅助说明明确后续批次实现。

## 验证

- 组件测试验证四个导航页面、指定文案、临时主题切换和浮窗禁用保存按钮。
- TypeScript 与 Vite 构建验证前端工程。
- Electron 启动烟雾测试验证应用能创建并正常关闭窗口；若桌面会话限制可视化检查，则明确记录限制。
- 静态范围扫描确认没有数据、图片、搜索、外部 API 或后续 Batch 模块。
