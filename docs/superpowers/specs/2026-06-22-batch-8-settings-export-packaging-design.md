# Batch 8 设置、导出与打包设计

## 目标与范围

在现有 Electron + React + TypeScript + Vite 技术栈上完成第一版收尾能力：真实设置页、主题与快捷键持久化、默认分类、数据目录查看、Markdown/JSON 导出、本地备份与覆盖式恢复、首次引导、关于区域以及 Windows 离线安装包配置。

本设计只覆盖 Batch 8。不得创建 Batch 9，不增加 AI、行情、荐股、交易、券商、账号、社区、云同步、手机端、OCR、独立图片库、人工总结、自动更新或远程上传能力。

## 总体架构

Electron 主进程继续作为唯一的本地文件系统边界。新增聚焦的设置服务与数据维护服务，通过最小 IPC 和现有 `contextBridge` 暴露给 React；渲染进程不获得 Node.js 或任意路径访问能力。

设置单独存储在 `app.getPath("userData")/app-data/settings.json`，笔记继续存储于 `notes.json`，附件继续位于 `attachments/`。这样设置、业务数据和附件职责清晰，且可直接组成备份目录。

## 设置模型

首版设置结构包含：

- `schemaVersion`：设置格式版本；
- `theme`：`light`、`dark` 或 `system`；
- `floatingShortcut`：单一浮窗全局快捷键，默认 `Alt+J`；
- `defaultCategoryId`：默认分类 ID 或 `null`；
- `onboardingDismissed`：是否已关闭首次引导。

设置服务使用临时文件后替换的原子写入方式。缺失文件自动创建；无效字段回退到默认值；默认分类不存在时回退到未分类。设置读取和修复均不改变笔记数据。

## 设置页面与主题

设置页沿用现有卡片、间距、圆角和颜色变量，按以下六区展示：外观设置、浮窗设置、默认记录设置、数据位置、导出与备份、关于项目。页面保持单列或响应式双列，不引入新的设计系统和复杂动画。

主题切换后立即更新应用根节点的 `data-theme`。主进程保存主题并广播设置变更，使主窗口与浮窗同步；`system` 通过 Electron/浏览器原生颜色方案检测解析，不引入主题库。应用启动时先读取设置再应用主题。

## 快捷键

主进程维护当前已注册快捷键。修改时先校验非空格式，再注销旧快捷键并尝试注册新快捷键；若失败，则恢复旧快捷键并返回冲突或无效提示。仅在新快捷键注册成功后持久化。

设置页显示当前快捷键，提供编辑、保存和恢复 `Alt+J`。应用启动时读取持久化值并注册；注册失败时继续保留主窗口中的浮窗显隐入口，并显示降级提示。

## 默认分类

设置页只读取本地 Category 列表并选择默认值，不提供分类增删改。浮窗每次加载组织数据时同时读取设置；当前草稿没有有效分类时选中默认分类。

若默认分类已删除，设置服务或设置读取协调层将其规范化为 `null`，浮窗显示“未分类”。这满足安全回退，同时不重新创建或修改分类。

## 数据位置

设置页显示 `app-data` 绝对路径，提供“打开数据目录”。主进程使用 Electron `shell.openPath`，不允许渲染进程提交任意目标路径，也不实现数据路径迁移、网络路径或云路径。

## Markdown 导出

用户通过系统目录选择器选择父目录。主进程创建 `export-markdown-YYYYMMDD-HHmmss/`，其中每条笔记生成一个 `notes/<日期>-<安全标题>.md`。文件名移除 Windows 非法字符、保留合理长度，并用数字后缀解决重名。

Markdown 包含标题、创建时间、更新时间、分类、标签、股票名称、股票代码、正文和图片引用。附件复制到 `attachments/note_<id>/`，Markdown 使用从 `notes/` 指向附件的相对路径。单个附件复制失败则整次导出失败并返回简洁错误，不上传数据。

## JSON 导出

用户选择父目录后创建 `export-json-YYYYMMDD-HHmmss/`。`data.json` 包含 `schemaVersion`、`notes`、`categories`、`tags`、`attachments` 和 `settings`；附件记录中的路径转换为导出包内相对路径，并将附件复制到 `attachments/`。不写图片二进制、不压缩、不加密、不上传。

## 备份

一键备份通过系统目录选择器创建 `backup-YYYYMMDD-HHmmss/`，包含 `notes.json`、`settings.json` 和 `attachments/`。缺失的附件目录按空目录处理；数据文件必须先成功读取和校验。备份采用文件夹复制，不引入压缩依赖。

## 恢复与回滚

用户选择一个备份目录后，主进程先校验 `notes.json`、`settings.json` 和附件结构。界面明确提示这是覆盖式恢复，并要求确认。

确认后，主进程先在应用数据目录旁创建时间戳安全备份；再将选择的备份复制到临时恢复目录，完整校验后替换正式文件。任何替换失败都使用安全备份回滚。成功后重新加载设置与数据、重新注册快捷键并广播刷新；若运行环境仍需重启，则明确提示用户。

## 首次引导与关于项目

首次引导是主窗口中的简短可关闭面板，说明浮窗文字记录、图片粘贴或拖入、分类和标签、知识库搜索整理、定期导出备份。关闭后持久化 `onboardingDismissed`，不包含登录、AI 或行情引导。

关于区域显示项目名称、版本 `0.1.0`、定位“Windows 桌面浮窗版 A 股学习笔记软件”、本地保存说明，以及免责声明：“本软件仅用于学习笔记和个人经验记录，不提供投资建议，不推荐股票，不进行自动交易。”

## Windows 打包

采用 Electron 生态常用的 `electron-builder`，生成 Windows x64 NSIS 离线安装包。`package.json` 提供清晰的 `dist:win` 命令，构建前执行类型检查、测试和 Vite 生产构建。配置不包含自动更新、遥测、联网安装器或远程发布。

## 错误处理与安全边界

- 所有文件操作在主进程完成，IPC 参数只传受控设置值或系统选择器结果；
- 快捷键注册失败保留旧设置和旧注册；
- 导出与备份写入独立时间戳目录，失败不修改应用数据；
- 恢复先校验、再安全备份、后替换，失败回滚；
- UI 使用简洁、可操作的中文反馈，不显示底层堆栈；
- 不访问网络，不上传任何内容。

## 测试与验收

渲染层测试覆盖设置区块、主题即时切换、默认分类、快捷键成功和失败提示、导出备份反馈、恢复确认、首次引导关闭。最后运行完整测试、类型检查、生产构建与 Windows 打包，并依据 `docs/03_ACCEPTANCE_CHECKLIST.md` 做 Batch 1—8 回归核对。
