# Batch 计划

Batch 必须按顺序执行。每个 Batch 仅在用户确认前一 Batch 通过后开始；以下只冻结目标和边界，不展开实现细节。

## Batch 1：项目骨架 + 现代简洁 UI 基础

**目标：** 建立可运行的 Windows 桌面应用骨架和统一的现代简洁 UI 基础。

**边界：** 只处理应用骨架和视觉基础，不实现浮窗业务、持久化或笔记功能。

## Batch 2：桌面浮窗核心功能

**目标：** 建立小、轻、低干扰、可折叠的 Windows 桌面浮窗基础交互。

**边界：** 只实现浮窗本身的核心行为，不实现持久化、完整图文记录或知识库。

## Batch 3：本地数据存储

**目标：** 建立本地优先的数据保存、读取和基础迁移边界。

**边界：** 只处理本地数据能力，不接入云端、登录系统或外部数据服务。

## Batch 4：图文记录功能

**目标：** 支持文字笔记以及图片粘贴、拖入和本地附件关联。

**边界：** 图片仅作为笔记附件，不建设独立图片库。

## Batch 5：分类与标签系统

**目标：** 支持分类、标签和股票名称/代码字段的维护与关联。

**边界：** 只做用户手动组织，不加入自动整理或人工总结模块。

## Batch 6：主窗口知识库页面

**目标：** 建立左侧分类、中间列表、右侧详情的主窗口知识库页面。

**边界：** 只展示和管理本地笔记，不接入行情或交易信息。

## Batch 7：搜索与筛选

**目标：** 支持对本地笔记按文本、分类、标签和股票名称/代码进行搜索与筛选。

**边界：** 只检索本地内容，不进行外部搜索、荐股或智能推荐。

## Batch 8：设置、导出、打包

**目标：** 完成设置、导出、备份、恢复和 Windows 打包，使第一版可交付。

**边界：** 不加入云同步、账户体系或第一版范围外的扩展能力。

## 全局禁止项

所有 Batch 均明确禁止：AI 自动整理或每日复盘、行情功能、荐股、券商同步、自动交易、独立图片库、人工总结模块。Batch 0 完成后不得自动进入 Batch 1。
